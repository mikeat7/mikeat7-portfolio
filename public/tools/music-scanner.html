Music Library Scanner & Builder

Two command-line tools that rescued a thirty-year music collection — and can rescue yours. The Scanner diagnoses every MP3 and WAV in your library: corruption, songs that cut off early, duplicates, ringtone-length fragments, even live recordings with spoken intros. The Builder then copies only the healthy files into a clean, alphabetized library. Your originals are never touched.

The Story (and the proof it works)

This suite was built to answer one question: can a script detect whether an MP3 ends prematurely? It was then run against a real collection — 1,338 vinyl-rip WAVs and 4,706 Napster-era MP3s. The diagnosis:

35%
of Napster-era MP3s truncated
53%
duplicates
0
defects in carefully-made vinyl rips

The strategy that emerged: don't operate on the sick patient — clone the healthy one. The Builder extracts every file that passes integrity checks, runs 90 seconds or longer, and is the best copy among its duplicates, into a fresh Artist - Title library.

Setup (once)

You need Python 3 (python.org/downloads — check "Add Python to PATH" during install). For the live-recording detector only, also run:

pip install librosa

Using the Scanner

# Report only — nothing is moved
python music_scanner_v2.py "C:\Music\My Music"

# Move corrupt/truncated files to quarantine
python music_scanner_v2.py "C:\Music\My Music" --quarantine "C:\Music_Bad"

# Sort files under 90 seconds (ringtones, fragments)
python music_scanner_v2.py "C:\Music\My Music" --short "C:\Music_Short"

# Flag live recordings (spoken intros / applause) for review
python music_scanner_v2.py "C:\Music\My Music" --live "C:\Music_Live_Review"

Each run saves a CSV report (open in Excel, sort the status column) and a plain-text summary.

StatusMeaning
OKHealthy, structurally complete.
TRUNCATEDEnds before it should — the song cuts off. Report shows what % is present.
CORRUPTBroken frame structure; often unplayable.
READ_ERRORWindows couldn't open the file (locked or permissions).

Using the Builder

# Build a clean library from one or more source folders
python library_builder.py "C:\Music\albums" "C:\Music\My Music" --output "C:\Music_Clean"

Qualifying rules: status OK · duration ≥ 90 s · longest copy wins among duplicates. Output: one flat folder, Artist - Song Title.ext, alphabetized ignoring "The/A/An". Originals untouched.

Privacy: everything runs on your own computer. No internet connection is used (except the one-time librosa install), nothing is uploaded, and nothing is deleted — files are only ever copied or moved to folders you name.
Honest accuracy note: the live-recording detector is a pre-filter, not a judge — roughly 80% accurate on applause, 70% on spoken intros. It moves candidates to a review folder for your ears to decide. Soft acoustic intros can trigger false positives.
